# 重点网站职务信息爬取策略

本文档总结了各重点政务网站的职务信息获取方法，供 `inspect.py` 脚本及 AI 辅助判断使用。

---

## 1. 广东省人民政府 (gd.gov.cn)

### 页面特征
- 领导人员信息页通常在"政务公开 > 领导信息"栏目
- 职务信息**内嵌在个人简历描述中**，不单独列出
- 简历通常包含：出生年月、籍贯、学历、历任职务、现任职务

### 爬取策略
1. 查找 `.leader-info`、`.person-info`、`.resume`、`.intro`、`.profile` 等 CSS 类的容器
2. 在简历文本中查找 **"现任"** 或 **"担任"** 关键词后跟随的职务短语
3. 简历末尾通常是现任职务，从后往前扫描职务关键词短语
4. 结合姓名定位（注意两字姓名可能有空格）

### 注意事项
- 职务信息在连续的简介段落中，需要整段读取后提取
- 不要只取第一个职务关键词，要找最近的"现任"描述
- 页面可能有多位领导信息，要通过姓名精确定位

---

## 2. 广东省政协委员会 (gdszx.gov.cn)

### 页面特征
- 每位委员有独立详情页
- 职务信息在页面的 **"个人简历"** 板块中
- 格式：列出历任职务，最后一条通常为现任职务

### 爬取策略
1. 优先查找含"个人简历"文本的标题标签（h2/h3/h4/div）
2. 取"个人简历"标题之后的兄弟节点内容
3. 在简历文本的最后几行中找包含职务关键词的短语（通常是现任职务）
4. 也可在父节点全文中使用"现任"/"担任"关键词定位

### 页面结构示例
```
个人简历
20xx年xx月，担任xxx职务
20xx年xx月，任xxx职务
现任广东省政协委员、xxx职务
```

### 注意事项
- 简历按时间顺序排列，**最后一条**通常是现任职务
- 部分委员职务较多，需扫描完整简历再取最后任职

---

## 3. 广东省人民代表大会 (gdpc.gov.cn)

### 页面特征
- 代表/领导信息页有人员头像
- 职务信息在 **头像下方的描述文字** 中
- 通常格式简洁，直接列出现任职务

### 爬取策略
1. 查找 img 标签，alt 属性包含人员姓名，取其父容器文本
2. 查找 `.leader-info`、`.person-desc`、`.member-info`、`.leader-desc` 等 CSS 类
3. 查找 "职务：姓名" 或 "职务名 姓名" 的格式（`re` 匹配）
4. 头像容器通常紧跟着职务简介

### 页面结构示例
```html
<div class="leader-info">
  <img src="photo.jpg" alt="张三" />
  <p>广东省人大常委会副主任</p>
  <p>张三</p>
</div>
```

### 注意事项
- 职务在头像下方，姓名在职务下方（职务在前，姓名在后）
- 要先定位姓名所在容器，再向上查找职务

## 4. 通用政务网站策略

### "职务：姓名" 格式
部分网站按以下格式展示领导信息：

```
主任：张三
副主任：李四
委员：王五
```

爬取时需用正则识别此格式：
```python
pattern = r'([^\n：:，]{2,20})[：:]\s*(姓名)'
```

### 通用文本提取顺序
1. 先找 CSS 类名含 `leader`、`person`、`member`、`intro`、`profile`、`resume` 的容器
2. 在容器内搜索"现任"、"担任"关键词
3. 找姓名前后100字符窗口，提取含职务关键词的最短短语
4. 全文从后往前扫描职务关键词行

---

## 5. 职务规范化规则

| 规则 | 说明 | 示例 |
|------|------|------|
| 去除括号内容 | 括号内通常为附加说明 | `主任（正厅级）` → `主任` |
| 去除"广东"前缀 | 仅"广东"前缀可忽略 | `广东省政协主席` → `省政协主席` 但 `广州市长` 不处理 |
| 忽略两字姓名空格 | 部分排版会在两字姓名中加空格 | `张 三` → `张三` |
| 不忽略其他地名前缀 | 其他城市/地区前缀保留 | `广州市市长` 保持原样 |

## 7. F列结果说明

| 值 | 触发条件 |
|----|---------|
| `一致` | 网页职务与D列职务规范化后完全匹配 |
| `不一致` | 网页职务与D列职务不匹配，或页面加载失败 |
| `无网址` | E列为空或无效 |
