# 重点用户职务信息巡检 (job-inspector)

## 功能概述

根据 Excel 表格中的网址批量爬取重点用户的职务信息，与表格中记录的职务进行智能比对，将结果回写到 Excel 指定列。

## Excel 表格列说明

| 列 | 含义 | 说明 |
|----|------|------|
| A | 姓名 | 待查人员姓名 |
| B | 账号状态 | 包含"暂无粤政易账号"时跳过本行 |
| D | 记录职务 | Excel 中已记录的职务信息（用于比对） |
| E | 网址 | 人员信息页面网址（为空则跳过） |
| F | **比对结果**（写入） | 一致 / 不一致 / 无网址 / 无帐号 |
| G | **网页职务**（写入） | 从网页爬取到的职务信息原文 |

## 跳过规则

- E 列无网址 → F 列写 `无网址`
- B 列含"暂无粤政易账号" → F 列写 `无帐号`
- 以上两条优先于爬取操作

## 比对规则

比对前对职务字符串进行规范化处理：

1. **去除括号内容**：`主任（正厅级）` → `主任`
2. **去除"广东"前缀**：`广东省政协主席` → `省政协主席`（仅"广东"前缀可忽略）
3. **去除空格**：处理两字姓名排版空格问题
4. 规范化后完全匹配 → `一致`，否则 → `不一致`

## 使用方式

### 方式一：直接运行脚本（推荐）

```bash
# 安装依赖（首次运行自动安装）
pip install openpyxl requests beautifulsoup4

# 运行巡检
python3 ~/.workbuddy/skills/job-inspector/scripts/inspect.py /path/to/your-file.xlsx

# 可选：调整请求间隔（默认1秒，建议1-3秒）
python3 ~/.workbuddy/skills/job-inspector/scripts/inspect.py /path/to/your-file.xlsx --delay 2
```

脚本会：
- 逐行读取数据，跳过无效行
- 按域名自动选择专项爬取策略
- 每行处理后**实时保存进度**，中断后可重新运行（已有结果的行会被覆盖更新）
- 控制台输出每行处理状态

### 方式二：AI 辅助逐行处理

当脚本无法自动识别职务时，可让 AI 手动处理：

1. 读取 `references/site-strategies.md` 了解各网站爬取策略
2. 打开网页，按策略定位职务信息
3. 按规范化规则比对后填写 F/G 列

## 重点网站专项策略

各网站的详细爬取策略见 `references/site-strategies.md`，摘要如下：

| 网站 | 职务位置 | 策略要点 |
|------|---------|---------|
| gd.gov.cn（省政府） | 个人简历描述中 | 找"现任"/"担任"关键词，简历末尾取现任职务 |
| gdszx.gov.cn（省政协） | "个人简历"板块 | 定位"个人简历"标题，取后续内容最后一条职务 |
| gdpc.gov.cn（省人大） | 头像下方描述 | 找头像容器或"职务：姓名"格式 |
| 其他网站 | 通用扫描 | CSS 类名定位 → "现任"/"担任"关键词 → 姓名窗口搜索 |

## 注意事项

1. **完整扫描页面**：不要只看页面前半段，要读取整个页面后再判断
2. **两字姓名空格**：部分网站排版会给两字姓名加空格，识别时需去除
3. **职务在姓名前**：部分网站按"职务：姓名"展示，要识别此格式
4. **括号内容忽略**：职务中的括号内容（如级别说明）不作为比对依据
5. **仅忽略"广东"前缀**：`广东XX` 可去掉"广东"比对，`广州XX` 等其他地名不可忽略
6. **实时保存**：脚本每处理一行即保存一次，网络中断不丢失进度

